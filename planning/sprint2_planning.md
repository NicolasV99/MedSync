# Sprint 2 Planning & QA Framework Design

## Initial QA Test Plan
- **Backend Testing Framework:** Jest/PyTest for unit testing and backend logic validation.
- **End-to-End (E2E) Testing Framework:** Cypress for mapping and testing complete user flows.

## Sprint 2 User Stories & Acceptance Criteria
I have established a comprehensive Given-When-Then acceptance criteria matrix for the core clinical and administrative features.

### Feature: Patient CRUD Operations
* **Create:** As an administrator, I want to add new patient profiles so their medical records can be tracked.
* **Read:** As a clinician, I want to search and view patient history to provide accurate care.
* **Update:** As an administrative user, I want to edit patient details to keep records up to date.
* **Delete:** As an authorized manager, I want to archive or remove records to maintain data compliance.
